# What is MQTT


MQTT stands for Message Queuing Telemetry Transport, it is a simple communication protocal for multiple devices.

## Point of View:

It uses a publish-subscribe messaging pattern, which means that any source of information (such as a sensor) can publish its data, and then any client can subscribe to that data.


## Basic Concept of MQTT:

### Publish/Subscribe


<img src="/images/posts/MQTT/publishSubscribe.png" height="200" width="600"> 

- Device 1 publishes on a topic.
- Device 2 is subscribing the same topic Device 1 was publishing in,
- Device 2 receives the message.

Messages are information exchanged between your device, aka. data.

### Topics

- Interest for **incoming messages**
- Specify where you want to **publish**

Topics are represented with **strings** separated by slashes "/". Slahes indicate the *topic level*.


One example:
```
home/office/lamp
```


### Broker

- Receives all the messages
- Filters the messages
- Publishes the messages to all subscribed clients
